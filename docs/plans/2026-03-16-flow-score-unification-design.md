# Flow Score Unification — Design Document

**Date:** 2026-03-16
**Branch:** feature/dev-staging
**Status:** Approved

## Goal

Unify the existing RiskLevel (SAFE/MODERATE/HIGH) and exhaustion score into a single **Flow Score** with 4 levels: Equilibrado, Moderado, Intenso, Exaustivo. This simplifies the UX and gives doctors a single, clear metric to understand their workload.

## Approach: Refactor In-Place

Modify existing engines (WorkloadEngine, RiskEngine) to produce Flow Score instead of RiskLevel. Migrate Prisma schema. Update all frontend pages.

---

## Backend Changes

### New Enum: FlowScore

Replaces `RiskLevel` (SAFE/MODERATE/HIGH):

```
EQUILIBRADO — balanced, sustainable workload
MODERADO    — manageable but approaching limits
INTENSO     — heavy workload, recovery needed
EXAUSTIVO   — dangerous overload, action required
```

### New Fatigue Weights

```
TWELVE_DAY      = 1.0
TWELVE_NIGHT    = 1.4
TWENTY_FOUR     = 2.2
TWENTY_FOUR_INV = 2.4
```

### Flow Score Rules

| Level       | Hours/week | Night shifts | Long shifts (24h) | Consecutive |
|-------------|------------|-------------|-------------------|-------------|
| Equilibrado | ≤48h       | ≤1          | ≤1                | ≤2          |
| Moderado    | 49-60h     | ≤2          | ≤2                | ≤3          |
| Intenso     | 61-72h     | ≥2          | multiple          | ≥3          |
| Exaustivo   | >72h       | ≥3          | consecutive long  | long chains |

Logic: worst level across all dimensions wins.
